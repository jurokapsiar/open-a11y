Select Behavior

Overview

This page is for testing different types of behaviors and the observations.

Currently this is testing the built-in browser based <select> control

States:

Open - this is when the popup is open and the state is placed on the select.

When the user invokes the button via pointer or keyboard

  • presses space or enter on the keyboard
  • the state is transitioned to open

Upon popup open:

  • Focus is transitioned to the currently selected option within the popup.
  • If necessary, the list is adjusted to ensure that the currently selected option is within view to the user
  • All other input events are still fired
  • The popup position direction is dependant on the location of the select within the

Closing the popup:

  • When the user:
    • engages with a pointer outside of the popup
    • When the user invokes an option with keyboard or point the currently selected value is update to the value of the option that was invoked.
    • user dismisses the popup by pressing the escape key
    • user dismisses the popup by tabbing to another element
  • the popup closes and the open state is removed from select

Misc observations:

  • Firefox & Chromium show 20 options. This will probably need to be adjusted since layout based adjustments can now impact options.
